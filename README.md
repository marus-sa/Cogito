[Проблемное поле]

У исполнителей-репетиторов: 
Когда новый репетитор приходит в проект, он сталкивается с непониманием, как устроена работа преподавателя в благотворительном проекте. Ему непонятен процесс связи с учениками, отправки видео-записей(репетиторы обязаны прикреплять видео-записи уроков и скрины начала и конца), возникает сложность в поиске инструментов для проведения занятий. В конечном итоге, ему необходимо использовать множество приложений, в которых ученики путаются, а именно: диск с материалами, гугл формы с дз(например, загруженные в классруме), а если репетитор решает не использовать приложения для этого, то появляется куча файлов, в которых тоже возникает путаница.

У исполнителей отдела интеграции:
Множество чатов, куда репетиторы кидают видео-записи и скрины, нет структуризации.

У учеников:
Ученик путается в приложениях или файлах, теряет необходимые материалы, некоторые ученики не обладают достаточной цифровой грамотностью, поэтому процесс "скачай файл с диска, открой, заполни, сохрани, переименуй, загрузи в форму" вызывает стресс и отнимает время от учебы. Также, для поддержания мотивации важно видеть процесс.

У родителей:
У учеников младших классов за процессом обучения следят родители, следовательно, с проблемами учеников, описанными выше, сталкиваются взрослые. 

Многим родителям важно видеть результат, поэтому они постоянно интересуются, как прошло занятие, но иногда они могут бояться нарушить границы, написав/позвонив слишком поздно. 


Маркеры проблемы:
- отсутствие единой экосистемы;
- избыток необходимых приложений для обучения;
- нехватка обратной связи о прошедшем занятии;
- сложность отправки записи занятий;
- сложность понимания процесса взаимодействия с наставником(наставник -  человек, которому отправляем записи занятий; это необходимо для контроля проведения занятий);
- отсутствие необходимого уровня цифровой грамотности

[Образ продукта]

Веб-приложение Cogito - это единая цифровая экосистема для образовательного волонтерства. Где все процессы - от первого знакомства с репетитором до анализа глобальной статистики проекта - происходят в одном месте, логично и удобно.

1. Регистрация 
1) пользователь открывает приложение
2) пользователь выбирает свою функциональную роль 
3) пользователь вводит свои данные (обязательно почту)
4) пользователь устанавливает пароль
5) пользователь проходит короткое обучение по навигации
6) Система запрашивает дополнительную информацию для анкеты:
у репетитора:
- Класс
- Предметы преподавания
- Удобное время для занятий
у ученика:
- Класс
- Необходимый предмет для изучения 
- Удобное время для занятий

2. Коммуникация между участниками
1) Открытие раздела "Чаты"
2) Просмотр списка доступных чатов (по ученикам, группам, проектам)
3) Выбор конкретного чата (например, с репетитором, родителем, координатором)
4) Просмотр истории переписки
5) Написание нового текстового сообщения
6) Прикрепление файлов к сообщению (документы, изображения)
7) Добавление новых участников в существующий чат

3. Постановка и отслеживание учебных целей
1) Открытие раздела "Мои цели"
2) Нажатие "Новая цель"
3) Формулировка цели (конкретная, измеримая, достижимая)
4) Установка срока выполнения цели
5) Разбивка цели на подзадачи и контрольные точки
6) Регулярное обновление прогресса по подзадачам(со стороны ученика и репетитора)
7) Просмотр прогресса и статистики достижения
8) Завершение цели, получение бейджа и мотивационного сообщения

4. Анализ успеваемости и прогресса (репетитор)
1) Добавление результатов работы на уроке
2) Ввод данных: тип работы, дата, баллы, комментарии
3) Отметка освоенных тем и зон роста
4) Автоматическое обновление графиков успеваемости
5) Просмотр динамики прогресса за выбранный период
6) Родитель просматривает статистику, по вопросам - коммуницимует с преподавателем

5. Отслеживание отработанных часов репетитором
1) Открытие раздела "Мои часы"
2) Выбор даты занятия
3) Отметка - занятие проведено
4) Прикрепление видео-записей и скринов
5) Добавление часов в итоговое число работы

- Неудачное решение на этапе проектирования - рефакторинг приложения
